General workflow for automated spoken corpus annotation of Latvian dialects

Dmitri Sitchinava1, Kirill Kozhanov2, Danguolė Kotryna Kapkan3, Ilja Seržant4 (University of Potsdam1,2,3,4, Potsdam Typology Lab2,4, Potsdam Slavic Variation Lab1,2,4, 2026)

Introduction

This document describes the procedure for creating a Universal Dependencies annotated corpus of Latvian (including Latgalian, often treated as a separate idiom) dialectal spoken texts. The corpus is designed as a linguistically annotated resource for the study of the vocabulary, grammar, and discourse structure.

The corpus may be used for research on lexical and grammatical variation in spoken dialectal texts. At the same time, the use of a unified token-based annotation format makes the resource suitable for comparative and typological studies, including studies that require interoperable morphosyntactic annotation across languages and dialects.

The corpus is maintained in two parallel and synchronized formats. The first format is CoNLL-U, which is used for Universal Dependencies publication, validation, and subsequent model training. The second format consists of annotation tiers in ELAN, where the text is synchronized with the original audio signal and where transcription, segmentation, pauses, speaker turns, and other oral-text features are preserved.

At the final stage, the morphosyntactic annotation produced and corrected in CoNLL-U, together with the normalized Latvian text, is copied back into ELAN as additional annotation tiers. This makes it possible to use both formats in parallel: CoNLL-U for UD-compatible morphosyntactic annotation and model training, and ELAN for audio-aligned corpus work and further linguistic analysis.

Most currently available Universal Dependencies treebanks are based on written language, and many existing parsing models have therefore been trained primarily on standard written syntax. Nevertheless, there is a growing line of UD corpora based on spoken language, including corpora derived from field recordings and oral narratives. The corpus is intended to contribute to this spoken-language direction within the UD framework.

ELAN

The annotation workflow starts from the audio recording and transcription. The WAV file and the primary transcription are imported into ELAN, where the transcription is synchronized with the audio signal. Speaker turns, pauses, hesitation phenomena, interrupted words, unclear fragments, and interviewer/interviewee boundaries are marked at this stage. The result is an ELAN file in which the oral text is time-aligned and segmented into analyzable discourse units.

Normalization/Bootstrapping

The next step is word-by-word normalization of the vernacular text into Standard Latvian orthography and morphology. This normalization is produced through the OpenAI API under strict constraints: the number of words must remain identical, the order of words must not be changed, no words may be inserted or deleted, and non-cognate vernacular roots must be kept rather than translated into unrelated Standard Latvian lexical equivalents. The normalization adds sentence boundaries, punctuation, and capital letters, because the downstream morphosyntactic models are trained on Standard Written Latvian rather than on dialectal speech.

This Latvian normalization is used as a bootstrapping procedure. Latvian/Latgalian dialectal speech do not yet have a large Universal Dependencies treebank suitable for direct parser training. Universal Dependencies currently has Latvian resources based on the Latvian LVTB treebank, and only a small proof-of-concept Latgalian resource, UD_Latgalian-Cairo, so Standard Latvian is used as the closest available computational bridge.

In parallel, an English translation of the text is produced through the OpenAI API. The translation is aligned at the phrase or sentence level and is later inserted into the CoNLL-U file as a comment before the corresponding phrase. This provides semantic access to the corpus and supports compatibility with broader dialectal corpus infrastructures such as DiBaSCo.

Annotation

A Python conversion script (draft version) then brings the layers together. It takes the normalized Latvian sentences as input for automatic morphosyntactic analysis with Stanza, using the Latvian model distributed as stanfordnlp/stanza-lv; in practice this is invoked as stanza.Pipeline("lv"), with the LVTB-based tokenizer/POS/lemma/dependency models. The Stanza model package includes Latvian tokenize, pos, lemma, and depparse components, with LVTB model files such as lvtb.pt and lvtb_nocharlm.pt.

The script enforces the sentence boundaries already introduced during normalization. Stanza is therefore used for morphological and syntactic analysis, but not allowed to re-segment or merge sentences. After annotation, the script substitutes the original, non-normalized token flow back into the FORM column, while retaining the Latvian-based lemma, UPOS, morphological features, heads, and dependency relations. The normalized Latvian form is preserved in metadata, so that each token line contains both the original dialectal form and the normalized Latvian analysis base.

The CoNLL-U output also includes phrase-level English translations as comments, for example before each sentence or phrase block. At a later stage, rule-based procedures may add lemma translations to token-level comments, so that each dialectal token can be connected both to a Latvian lemma and to an English gloss or translation.

Expert correction and model training

The automatically produced file is then manually corrected in two stages. First, the morphological annotation is corrected in a text editor. This includes correcting POS tags, lemmas, morphological features, unclear forms, particles, discourse markers, and dialect-specific forms. Second, the syntactic annotation is corrected using tree-visualization tools such as ArboratorGrew. This step corrects heads, dependency labels, coordination, ellipsis, reparanda, disfluencies, and clause-level structure.

A major methodological problem is that the Latvian model is trained on standard written syntax, while the target material is dialectal and spoken. The model therefore tends to mishandle features typical of spoken language: false starts, repetitions, repairs, interrupted constructions, discourse markers, elliptical answers, interviewer backchannels, and reparandum-like material. These phenomena require explicit annotation decisions and substantial manual correction.

The use of Latvian lemmas has a practical advantage: it makes the resulting corpus interoperable with larger Latvian dialectal and vernacular corpora. At the same time, the English phrase translations and future lemma translations support broader cross-linguistic compatibility, especially with larger dialectal corpus frameworks such as DiBaSCo.

The first target is a test corpus of more than 15,000 tokens. After internal validation, this corpus is forwarded to the Universal Dependencies team, in collaboration with the Prague ÚFAL infrastructure. The tagset and annotation decisions are documented, the CoNLL-U files are validated, and the treebank is prepared for publication at universaldependencies.org. Once the corrected treebank(s) are accepted and published, a dedicated model can be trained on it and then used for further annotation of the oral texts directly, reducing the dependence on Standard Latvian bootstrapping. Universal Dependencies provides a common framework for word segmentation, lemmas, UPOS tags, morphological features, and dependency syntax, while UDPipe/UDPipe2 infrastructure is widely used for UD-based parsing and model training.
